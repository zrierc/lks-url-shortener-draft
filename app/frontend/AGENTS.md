# Service: frontend (`frontend/`)

## Runtime

- **Build tool**: Vite
- **Framework**: React 19 + TypeScript
- **Router**: TanStack Router (file-based)
- **Data fetching**: TanStack Query v5
- **Charts**: Recharts
- **Styling**: TailwindCSS v4.2
- **Components**: ShadCN UI
- **Form validation**: Zod
- **Linter & Formatter**: Biome
- **Server**: nginx:alpine (serves static build)
- **Port**: 80
- **Container**: `oven/bun:1` (build) → `nginx:alpine` (runtime)

## Pages

### `/` — Landing Page (`index.tsx`)

- Static page — no data fetching
- Hero section with app name, tagline, and description
- Prominent CTA button → navigates to `/shortener`
- Clean, minimal design using TailwindCSS + ShadCN Card/Button
- No auth, no forms

### `/shortener` — Shorten Form (`shortener.tsx`)

- URL input field with Zod validation (must be valid URL)
- Submit button → `POST /api/shorten`
- On success: display the short link in a styled result card
- Copy-to-clipboard button next to the short link
- Loading state during submission
- Error state if the API call fails
- TanStack Query mutation for the POST request

### `/stats` — Stats Dashboard (`stats.tsx`)

- Short code lookup form — text input + submit
- On submit: fetch `GET /api/stats/:code`
- Display: click count, last clicked timestamp
- 4 charts rendered with Recharts:
  - **Clicks over time** — LineChart, x-axis: date, y-axis: count
  - **Device breakdown** — PieChart/Donut, segments: desktop/mobile/tablet/bot
  - **OS breakdown** — BarChart, x-axis: OS name, y-axis: count
  - **Browser breakdown** — BarChart, x-axis: browser name, y-axis: count
- Leaderboard section: fetch `GET /api/stats` → top 10 list
- Loading skeletons while data is fetching
- Empty state when no stats are available for a code

## API Client (`src/api/client.ts`)

All API calls are centralized here. Uses native `fetch`.
TanStack Query hooks consume these functions.

```typescript
const API_BASE = "/api"; // nginx proxies /api/* to ALB

export const apiClient = {
  shorten: async (url: string): Promise<{ code: string; short_url: string; original_url: string; created_at: string }> => {
    const res = await fetch(`${API_BASE}/shorten`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ url }),
    });
    const data = await res.json();
    if (!data.success) throw new Error(data.error);
    return data.data;
  },

  getStats: async (code: string): Promise<StatsResponse> => {
    const res = await fetch(`${API_BASE}/stats/${code}`);
    const data = await res.json();
    if (!data.success) throw new Error(data.error);
    return data.data;
  },

  getLeaderboard: async (): Promise<LeaderboardEntry[]> => {
    const res = await fetch(`${API_BASE}/stats`);
    const data = await res.json();
    if (!data.success) throw new Error(data.error);
    return data.data;
  },
};
```

## nginx.conf — Reverse Proxy

The frontend container serves the static Vite build and proxies `/api/*`
to the ALB. This is the ONLY place the ALB DNS appears in the app repo.

```nginx
server {
    listen 80;
    server_name _;
    root /usr/share/nginx/html;
    index index.html;

    # Proxy API requests to ALB
    location /api/ {
        proxy_pass          http://<ALB_DNS>/api/;
        proxy_http_version  1.1;
        proxy_set_header    Host              $host;
        proxy_set_header    X-Real-IP         $remote_addr;
        proxy_set_header    X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header    X-Forwarded-Proto $scheme;
    }

    # SPA fallback — all non-API routes serve index.html
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

`<ALB_DNS>` is filled after the first Terraform apply outputs `alb_dns`.
The value is the raw ALB DNS name (e.g. `lks-url-alb-123456.us-east-1.elb.amazonaws.com`).

## Design Principles

- Mobile-first responsive layout via TailwindCSS
- Use ShadCN `Card`, `Button`, `Input`, `Skeleton`, `Badge` components
- Loading states with `Skeleton` components — no layout shift
- Error states with a visible error message and retry option
- Empty states: friendly message when no data is found
- Color palette consistent across all 3 pages
