# Service: analytics-service (`analytics/`)

## Runtime

- **Runtime**: Bun latest stable
- **Framework**: Hono + TypeScript
- **ORM**: DrizzleORM (`drizzle-orm`, `drizzle-kit`)
- **Validation**: Zod
- **API Docs**: Swagger / OpenAPI via `@hono/zod-openapi`
- **IP Geolocation**: `geoip-lite`
- **UA Parsing**: `ua-parser-js`
- **Port**: 3001
- **Container**: `oven/bun:1` (build) → `gcr.io/distroless/base-debian12` (runtime)

## SQS Consumer — Polling Loop

The consumer starts when the app boots and runs indefinitely.
It must NOT block the Hono HTTP server — start it as a detached async loop.

### Polling Strategy

```typescript
// src/consumer/index.ts
export async function startConsumer() {
  console.log("[consumer] Starting SQS consumer...");

  while (true) {
    try {
      const messages = await pollMessages(); // long poll, 20s wait, up to 10 messages
      for (const message of messages) {
        await processMessage(message);
        await deleteMessage(message.ReceiptHandle!);
      }
    } catch (err) {
      console.error("[consumer] Error in polling loop:", err);
      await Bun.sleep(5000); // back off on error
    }
  }
}
```

### SQS Long Poll Parameters

```typescript
{
  QueueUrl:            env.SQS_URL,
  MaxNumberOfMessages: 10,
  WaitTimeSeconds:     20,   // long polling — reduces empty receives
  VisibilityTimeout:   30,   // matches infra config
}
```

### Message Processing — per ClickEvent

```
1. Parse message body (JSON) → validate with Zod ClickEvent schema
2. Resolve IP → { country, city } via geoip-lite
3. Parse user_agent → { device_type, os, browser } via ua-parser-js
4. Upsert url_stats:
     INSERT INTO url_stats (code, click_count, last_clicked)
     VALUES ($code, 1, NOW())
     ON CONFLICT (code) DO UPDATE SET
       click_count  = url_stats.click_count + 1,
       last_clicked = NOW()
5. Insert click_log:
     INSERT INTO click_log
       (code, clicked_at, ip, country, city, device_type, os, browser, referrer)
     VALUES (...)
6. Delete message from SQS
```

### ClickEvent Zod Schema

```typescript
const clickEventSchema = z.object({
  code: z.string(),
  clicked_at: z.string().datetime(),
  ip: z.string(),
  user_agent: z.string(),
  referrer: z.string(),
});
type ClickEvent = z.infer<typeof clickEventSchema>;
```

## GET /api/stats/:code — Response Shape

```typescript
type StatsResponse = {
  code: string;
  original_url: string; // from urls table — same RDS instance, SELECT original FROM urls WHERE code = ?
  click_count: number;
  last_clicked: string | null;
  // Chart data
  clicks_over_time: Array<{ date: string; count: number }>; // daily buckets
  by_device: Array<{ device_type: string; count: number }>;
  by_os: Array<{ os: string; count: number }>;
  by_browser: Array<{ browser: string; count: number }>;
  by_country: Array<{ country: string; count: number }>;
};
```

All chart arrays are aggregated from `click_log` with `GROUP BY`.
`original_url` is fetched from the `urls` table (owned by shortener-api but in the same RDS
instance). analytics-svc defines a read-only reference to the `urls` table in its schema index
and queries it directly — no HTTP call to shortener-api.

```json
  "services": {
    "postgres": { "status": "ok", "latency_ms": 3 },
    "sqs": { "status": "ok", "latency_ms": 18 }

```

SQS health check: call `GetQueueAttributes` with `QueueUrl` and measure latency.
If call fails: `{ "status": "error", "latency_ms": -1 }`.

## utils/geo.ts — geoip-lite Usage

```typescript
import geoip from "geoip-lite";

export function resolveLocation(ip: string): { country: string; city: string } {
  const geo = geoip.lookup(ip);
  return {
    country: geo?.country ?? "Unknown",
    city: geo?.city ?? "Unknown",
  };
}
```

## utils/ua-parser.ts — ua-parser-js Usage

```typescript
import { UAParser } from "ua-parser-js";

type ParsedUA = { device_type: string; os: string; browser: string };

export function parseUserAgent(ua: string): ParsedUA {
  const parser = new UAParser(ua);
  const result = parser.getResult();

  const deviceType = result.device.type ?? "desktop"; // mobile | tablet | desktop | bot
  const os = result.os.name ?? "Unknown";
  const browser = result.browser.name ?? "Unknown";

  return { device_type: deviceType, os, browser };
}
```

## index.ts — App Boot Sequence

```typescript
// Start HTTP server and SQS consumer concurrently
const app = createApp(); // register routes
startConsumer(); // non-blocking — starts the loop detached

export default {
  port: env.PORT,
  fetch: app.fetch,
};
```

The consumer must not block the server. Use fire-and-forget pattern:

```typescript
// Correct: consumer runs in background
startConsumer().catch(console.error);
```
